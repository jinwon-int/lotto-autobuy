# Lotto AutoBuy (로또 자동 구매)

동행복권 로또 6/45 자동 구매 + 당첨 확인 시스템.

## 현재 아키텍처 (2026-06-19~)

```
대교 (Seo S23 Ultra, Termux + Hermes)
├── Cron: 매주 토요일 18:00 KST → dhapi buy-lotto645 → 구매
└── Cron: 매주 토요일 21:00 KST → 당첨번호 조회 → 비교 → Telegram 알림
```

- **GitHub Actions는 폐기됨** — 대교가 직접 구매/확인 처리
- **비용**: $0/월 (Actions 예산 불필요)

## 구매 전략

현재 기본 전략은 **전략 C — 비인기 패턴 회피 + 분산 5조합**입니다.

| 항목 | 값 |
|------|------|
| 전략 | **분산 5조합** (1등 도달 확률 5배) |
| 목표 | “인생 역전 대박”을 노리되, TOP6/인기번호 몰빵의 공동당첨 분할 리스크 회피 |
| 주간 비용 | 5,000원 (5게임 × 1,000원) |
| 운영 경로 | 대교 Hermes cronjob에서 `dhapi buy-lotto645` 직접 실행 |

### 왜 TOP6 동일번호 ×5에서 바꿨나

A2AD 분석 결론:

1. 로또 6/45의 1등 확률은 고정 조합 기준 `1 / 8,145,060`입니다.
2. 동일번호 ×5는 적중 시 내 당첨 티켓이 5장이지만, 1등에 닿는 사건 확률은 여전히 `1 / 8,145,060`입니다.
3. 서로 다른 5조합은 1등에 닿는 사건 확률을 `5 / 8,145,060`으로 넓힙니다.
4. TOP6 최다출현 번호는 통계 추천/수동 구매자가 따라 고를 가능성이 있어, 1등 시 공동당첨 분할 리스크가 커질 수 있습니다.
5. “인생 역전” 목표에서도 20~30억원대 1등 한 장은 충분히 목표 범위에 들어오므로, 먼저 1등에 닿는 확률을 넓히는 편이 합리적입니다.

분할 모델의 핵심 조건:

```text
A_EV = P/N × 5/(5+kA)
B_EV = P/N × 5/(1+kB)
A가 B보다 유리하려면 kB > kA + 4
```

여기서 `kA`는 TOP6 동일 조합의 다른 당첨 티켓 수, `kB`는 분산 조합 쪽 다른 당첨 티켓 수입니다. TOP6 조합이 분산 조합보다 덜 붐빌 근거가 약하므로, TOP6 동일번호 ×5는 실전 우위가 약합니다.

## 기술 스택

- **Telegram** — 구매/당첨 결과 알림
